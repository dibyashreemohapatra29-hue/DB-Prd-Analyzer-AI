    ✨ Tips for Better PRDs

    • Define explicit acceptance criteria for every feature.
    • Specify error states alongside happy paths.
    • Include concrete numeric values — never "reasonable" or "quick".
    • Name every actor, system, and external dependency.

    How it Works

    A PRD analysis runs through nine stages from input to results.

    1

    Input

    You paste raw PRD or feature specification text. The minimum is 50 characters — too short and there's nothing meaningful to analyze.

    ↓
    2

    Validation

    The backend checks length, content type, and structure. Malformed or empty requests are rejected immediately with a clear error message.

    ↓
    3

    Rules Engine

    A local rule engine scans for known anti-patterns: vague words ("reasonable", "fast", "some"), missing actors, undefined inputs, and absent error handling.

    ↓
    4

    Prompt Building

    Rule findings and your PRD are assembled into a structured prompt. The format constrains the AI to return machine-readable JSON with typed issue objects.

    ↓
    5

    Groq LLM

    The prompt is sent to Groq's compound-beta-mini model — a fast inference model tuned for structured reasoning. Response latency is typically under 2 seconds.

    ↓
    6

    Response Parsing

    The raw model output is extracted from markdown code fences, parsed as JSON, and validated. Issues are typed (ambiguity, missing_logic, undefined_input) and capped at 3 per analysis.

    ↓
    7

    Confidence Score

    A score from 0–100% is computed from issue count: 0–1 issues → above 70% (Ready), 2–4 → 40–70% (Needs Improvement), 5+ → below 40% (Low Quality).

    ↓
    8

    Supabase Storage

    Each result is persisted to a Supabase PostgreSQL table (prd_analysis) with the full PRD text, issues, questions, score, and timestamp. This powers the History page.

    ↓
    9

    UI Results

    The frontend renders a summary card (score + status + top issues) and a tabbed detail view (Issues / Edge Cases / Assumptions / Questions) — no page reload required.
